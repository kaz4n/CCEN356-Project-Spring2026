## Script Specifications

### Script 1: `scripts/ssh_connect.py` — Netmiko SSH

**Purpose:** Connect to Cisco routers via SSH, execute show commands, print and return results.

**Behavior:**
- Connect to R1 at 192.168.1.1 using `netmiko.ConnectHandler` with `device_type='cisco_ios'`
- Credentials: username=`admin`, password=`admin123`, secret=`admin123`
- Call `connection.enable()` to enter privileged exec mode
- Execute these commands and store output in a dict keyed by command string:
  - `show ip interface brief`
  - `show ip route`
  - `show access-lists`
  - `show policy-map interface GigabitEthernet0/1`
- Print each command's output with a header
- Disconnect when done
- `if __name__ == "__main__":` calls the function with R1's IP

---

### Script 2: `scripts/capture_traffic.py` — Scapy Packet Capture

**Purpose:** Capture HTTP and HTTPS packets on the client's network interface for 60 seconds, log to CSV.

**Behavior:**
- Use `scapy.all.sniff()` on interface (detect with `ip a`, likely `eth0` or `ens33`)
- Timeout: 60 seconds
- Callback function filters packets that have both TCP and IP layers
- Only record packets where src_port or dst_port is 80 or 443
- Label as `"HTTP"` if port 80 involved, `"HTTPS"` if port 443
- Each entry: `timestamp`, `src_ip`, `dst_ip`, `src_port`, `dst_port`, `protocol`, `length`
- Save to `data/traffic_log.csv` using `csv.DictWriter`
- **Requires `sudo` to run**

**CSV columns:** `timestamp,src_ip,dst_ip,src_port,dst_port,protocol,length`

---

### Script 3: `scripts/performance_metrics.py` — HTTP vs HTTPS Benchmark

**Purpose:** Send 20 requests each to HTTP and HTTPS endpoints, measure and compare performance.

**Behavior:**
- Targets: `http://192.168.2.10` (HTTP) and `https://192.168.2.10` (HTTPS)
- 20 GET requests per target with `requests.get(url, timeout=10, verify=False)`
- Suppress urllib3 InsecureRequestWarning
- Measure per-request elapsed time in ms
- Sleep 0.2s between requests
- Metrics per protocol: `protocol`, `url`, `requests`, `errors`, `avg_ms`, `min_ms`, `max_ms`, `stdev_ms`, `throughput_kbps`, `error_rate_%`
- Save to `data/performance_results.csv`

---

### Script 4: `scripts/visualize_traffic.py` — Matplotlib Charts

**Purpose:** Read CSV data files and generate comparison charts as PNG images.

**Function 1: `plot_response_comparison()`** — reads `data/performance_results.csv`
- 1×3 subplots (15×5): avg response time bars, throughput bars, min/avg/max error bars
- Colors: HTTP=#2196F3, HTTPS=#4CAF50
- Save to `charts/performance_comparison.png` at 150 DPI

**Function 2: `plot_traffic_over_time()`** — reads `data/traffic_log.csv`
- 2×1 subplots (12×8): packet count by protocol, packet size distribution histogram
- Save to `charts/traffic_analysis.png` at 150 DPI

---

### Script 5: `scripts/dashboard.py` — Flask Live Dashboard

**Purpose:** Real-time web dashboard showing HTTP vs HTTPS response times.

**Behavior:**
- Flask on `0.0.0.0:5000`
- Background daemon thread polls both servers every 3s, stores last 100 samples
- `/` serves HTML with Chart.js line chart (dark theme, 2-column grid)
- `/api/metrics` returns JSON: `http_avg_ms`, `https_avg_ms`, `http_samples`, `https_samples`

## Server Specifications

### `server/secured_server.py` — Flask HTTPS Server (port 8443)

- SSL context using `cert.pem` and `key.pem`
- Security headers: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, HSTS, CSP
- Request logging to `server.log`
- Path traversal protection (reject `..`)
- Custom error handlers: 403, 404, 500
- Routes: `/` → `index.html`, `/show-something` → `show.html`

### `server/http_server.py` — Flask HTTP Server (port 80)

- Same routes, logging to `http_server.log`
- No SSL, no security headers (plain HTTP for comparison)

## Cisco Router Configurations

### R1 Configuration

```
hostname R1
enable secret cisco123
line console 0
 password cisco123
 login

interface GigabitEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
 description LAN_SIDE

interface GigabitEthernet0/1
 ip address 10.0.0.1 255.255.255.252
 no shutdown
 description WAN_TO_R2

ip route 192.168.2.0 255.255.255.0 10.0.0.2

ip domain-name lab.local
crypto key generate rsa modulus 2048
ip ssh version 2
username admin privilege 15 secret admin123
line vty 0 4
 transport input ssh
 login local

! ACL
ip access-list extended HTTP_HTTPS_ONLY
 permit tcp any any eq 80
 permit tcp any any eq 443
 permit tcp any eq 80 any established
 permit tcp any eq 443 any established
 permit icmp any any

interface GigabitEthernet0/1
 ip access-group HTTP_HTTPS_ONLY in

! QoS
class-map match-any HTTP_HTTPS
 match protocol http
 match protocol https

policy-map WEB_QOS
 class HTTP_HTTPS
  priority 1000
 class class-default
  fair-queue

interface GigabitEthernet0/1
 service-policy output WEB_QOS
```

## Wireshark Analysis Procedure

1. Start Wireshark on Monitor PC (SPAN port Gi0/10)
2. Filters: `tcp.port == 80 || tcp.port == 443`
3. Capture during script execution (60+ seconds)
4. Screenshots needed:
   - TCP 3-way handshake (SYN, SYN-ACK, ACK) for HTTP
   - TLS handshake (ClientHello, ServerHello, Certificate) for HTTPS
   - Data transfer frames
   - Statistics → IO Graph
   - Statistics → TCP Stream Graphs → Round Trip Time
5. Save as `.pcapng` for report appendix

## Key Decisions

- **Physical hardware** (not Packet Tracer) — Cisco 2901 routers, 2960 switch
- **Apache** on server for standard HTTP (80) / HTTPS (443); Flask servers supplementary (80 / 8443)
- **Self-signed SSL** certs — `verify=False` in Python scripts is acceptable
- **Scapy requires root/sudo** to capture packets
- **All Python scripts** run on client PCs (Linux)
- **Flask dashboard** (Script 5) is included
- **All code is fresh** — no pre-existing codebase
